## Entry 3 ŌĆö HMAC hardening (A-11, C-4, C-5) ŌĆö shared lib + v2 protocol

**Rationale:** Three findings cluster naturally: extract shared helper,
sign the order_id, widen skew. Doing all three in one sweep avoids touching
the callback handler twice.

**Scope:**
- Created `cards402/backend/src/lib/hmac.js` with `signCallback`,
  `verifyCallback`, and a hardened `safeEqHex` that rejects non-hex input.
  Mirrored into `vcc/api/src/lib/hmac.js`.
- **New v2 wire format:** HMAC over `${timestamp}.${orderId}.${rawBody}`,
  sent with a new `X-VCC-Order-Id` header. Defence in depth: the receiver
  also compares the header value to `req.body.order_id` and rejects with
  `order_id_mismatch` if they differ.
- **Transition path:** `verifyCallback` still accepts legacy v1 signatures
  (`${timestamp}.${rawBody}`) and reports which version validated in its
  result object so telemetry can track the rollout.
- **Skew window:** increased from 5 to 10 minutes (configurable per-call via
  `maxSkewMs`). NTP-sync guidance still pending ŌĆö rolled into the docs sweep.
- Added 27 unit tests in `backend/test/unit/hmac.test.js` (mirrored to
  `vcc/api/test/unit/hmac.test.js`) covering:
    v2 sign/verify happy path; v2 order_id swap rejection; v2 body tamper
    rejection; v1 backward compat (with and without spurious header);
    missing inputs; non-hex signature; stale and future timestamps;
    `maxSkewMs` override; injected `now`; `safeEqHex` edge cases.
- Updated `backend/src/api/vcc-callback.js` to use the new helper, map
  internal reasons to stable wire error strings, and emit
  `callback.rejected` bizEvent with the reason for ops metrics.
- Updated `vcc/api/src/fulfillment.js notifyCards402` to sign with the new
  helper and send the new header.
- Updated `cards402/ARCHITECTURE.md` ┬¦"Callback signature" to document the
  new v2 format, the 10min window, the `order_id_mismatch` rule, the shared
  lib location, and the v1 legacy fallback.
- Updated the existing `vcc-callback.test.js` replay-protection test to use
  a 15-min stale timestamp (was 6-min, now within the 10-min window).
- Extended `vcc-client.test.js` with v1 + v2 paths.

**Verification:** vcc 65/65 passing (was 42, +23 new hmac tests).
cards402 backend 188/188 passing (was 161, +27 new hmac tests).

**Findings closed:** A-11, C-4, C-5.
